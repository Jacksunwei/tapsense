# Implementation Plan: Decomposing TapSense into Sub-Projects

### Objective
Refactor the TapSense project into 4 distinct sub-projects to improve maintainability, boundary isolation, and support device adaptability.

---

### Target Architecture

1.  **`tapsense-core`**
    *   **Focus**: Core sensing and resampling logic.
    *   **Component**: The Sampler. Wraps raw detector API with a sampler that smoothly yields 200Hz sample points. Must use interpolation (linear) to adapt to all devices regardless of hardware rate.
2.  **`tapsense-cli`**
    *   **Focus**: Developer and test tools.
    *   **Feature: Data Collection**: Use heuristic detector method to record raw data out of the sampler as training data. Output: raw JSON (e.g., `desk_one_tap.jsonl`). Includes visual cue logic.
    *   **Feature: Model Tester**: Dynamically load trained models and test whether they work.
3.  **`tapsense-app`**
    *   **Focus**: Core menu app. Bundles the ML model.
4.  **`tapsense-vscode-ext`**
    *   **Focus**: VS Code extension. Utilizes signals sent by `tapsense-app`.
